# Performance Optimization Module

用于绩效差距、人才发展、继任、激励、留人与岗位匹配问题。

## 适用信号

- 人不差，但结果差
- 结果还行，但人要走
- 团队干得辛苦，绩效体系却没驱动正确行为
- 需要做继任、梯队、激励和角色匹配

## 核心看点

- 差距来源：能力、动力、环境、角色匹配
- 指标设计：是否真的驱动了正确行为
- 管理责任：是个人问题，还是 manager 和系统问题
- 保留风险：谁重要、为什么想走、什么条件可挽回
- 继任节奏：谁能代理、谁能培养、谁不该硬推

## 默认问题

1. 结果差是不会、不能，还是不想
2. 绩效指标是否鼓励了错误行为
3. 该留的人为什么没被激励，该走的人为什么没被处理
4. 这次是做纠偏、换岗、保留，还是退出安排

## 默认输出

- 绩效差距判断
- 关键原因排序
- 纠偏或保留方案
- 继任与梯队建议
- 激励机制调整建议

## 常见协作模块

- 涉及组织结构和流程影响 -> `organizational-effectiveness.md`
- 涉及情绪、信任和对话设计 -> `interpersonal-relations.md`
- 涉及组织变革配套 -> `change-management.md`

## 易错点

- 把系统问题都压到个人身上
- 用模糊绩效理由处理真实的信任问题
- 想留人却不给实际交换条件
