# HQ vs Local Power Game

这份模块处理外资、跨国公司、区域总部与本地团队之间的权力与执行博弈。

## 适用信号

- HQ 与中国区、本地团队、APAC、global function 存在拉扯
- 全球统一流程压不住本地现实，或本地团队以“特殊性”对冲 HQ 要求
- 用户关心授权、汇报线、资源、评价权和叙事控制权

## 核心判断

先把冲突拆成三层：

1. 原则冲突：目标是否一致
2. 路径冲突：方法、流程、节奏是否一致
3. 权力冲突：谁来解释、谁来批准、谁来背责

很多所谓“文化差异”，本质上是第 3 层。

## 默认打法

### 1. 别先争输赢，先争定义权

先统一问题定义：

- 是 compliance 问题
- 是 growth 问题
- 是 execution 问题
- 还是 org design 问题

谁定义问题，谁就更接近决定方案。

### 2. 把本地诉求翻译成 HQ 能接受的语言

少说：

- “中国情况特殊，所以必须例外。”

多说：

- “如果按当前流程执行，会造成 X 风险和 Y 损失。”
- “我们建议在原则不变的前提下，对执行路径做本地化调整。”

### 3. 别用情绪对全球流程，用证据和替代方案

至少准备：

- 本地业务影响
- 替代路径
- 需要的授权级别
- 不调整的代价

### 4. 同时管理三个对象

- 老板：需要安全感和可控性
- 平级函数：需要边界和尊重
- 本地团队：需要信号，知道你不是只会传达 HQ 指令

## 常见局面

### 1. HQ 想要统一，本地认为落不了地

推荐动作：

1. 接受统一原则，不直接挑战目标
2. 列出 2 个本地化备选方案
3. 明确试点期和验收指标

常用表达：

- “我们支持统一原则，但建议先用中国区试点版本验证。”
- “如果直接全量上线，预计会出现 A/B/C 三类损失。”

### 2. 本地负责人被架空

推荐动作：

1. 先确认是谁在切断信息和资源
2. 抢回两个关键权力：报告权、解释权
3. 通过定期书面 update 恢复存在感和可见度

常用表达：

- “如果结果由我承担，我需要同步掌握 X 和 Y 两项信息。”
- “为避免误解，后续这类事项我建议直接抄送到本地负责人链路。”

### 3. 全球函数越级指挥本地团队

推荐动作：

1. 不公开翻脸
2. 私下与 sponsor 对齐治理边界
3. 用 RACI 或 decision rights 固化边界

常用表达：

- “为了提速，我建议把建议权、决定权、执行权分开写清。”
- “我们欢迎专业支持，但本地执行链路需要保持单一。”

## 易错点

- 把“alignment”理解成“完全服从”
- 用民族情绪替代组织论证
- 一味讨好 HQ，导致本地失去信任
- 只说问题，不给可执行替代路径

## 安全边界

- 不建议伪造全球同意、伪造 local consensus
- 涉及跨境数据、调查、合规例外时，联动 `escalation-boundaries.md`
