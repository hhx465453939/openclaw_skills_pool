# Sensitive Phrasing Templates

这份模块用于敏感表达。原则是清楚、可执行、少留歧义，但不制造虚假事实。

## 总原则

- 先定目标：安抚、施压、拒绝、切割、通知、对齐，目标不同，句式不同
- 先定场景：口头、私聊、邮件、会上发言、对外口径
- 先定边界：哪些能确认，哪些不能确认
- 避免绝对化、人格化、情绪化、威胁化

## 口头表达骨架

### 1. 温和施压

- “这件事我们需要在本周内定方向，不能再停留在讨论层面。”
- “我理解你的顾虑，但现在更大的风险是继续拖。”

### 2. 明确拒绝

- “这个方案我现在不能同意，主要卡在 X 和 Y。”
- “方向可以继续讨论，但当前版本我不会签字。”

### 3. 安抚但不承诺

- “你的顾虑我收到了，我们会认真处理。”
- “今天先把事实和边界厘清，结论我不在这一刻仓促给。”

### 4. 切割责任

- “这部分不是我当前授权范围内的决定。”
- “我可以提供建议，但最终判断需要由 X 线负责确认。”

## 邮件表达骨架

### 1. 会后确认

```text
主题：关于 X 事项的会后确认

今天会议先确认三点：
1. ...
2. ...
3. ...

目前仍待补充：
- ...

如无异议，我方将按上述节奏推进下一步。
```

### 2. 风险提示

```text
基于当前信息，我需要明确提示两点风险：
1. ...
2. ...

这并不意味着事项不能继续推进，但在进入下一阶段前，建议先补齐：
- ...
```

### 3. 温和纠偏

```text
为避免后续执行理解偏差，我补充澄清如下：
- 原则上 ...
- 当前不包含 ...
- 后续需由 ... 确认
```

## 敏感场景模板

### 1. 劝退或降权前置沟通

- “这次沟通不是临时提醒，而是正式调整前的对齐。”
- “我希望我们把重点放在后续安排，而不是反复讨论过去的是非。”

### 2. 对上争授权

- “如果结果由我承担，我需要对 X 拥有明确决定权。”
- “我不反对目标，但当前授权设计不足以保障结果。”

### 3. 对平级设边界

- “我支持协同，但边界需要写清，否则后面很难追责和复盘。”
- “这部分欢迎建议，但最终解释权需要留在责任部门。”

### 4. 调查或举报通知

- “目前我们只做事实核查，不预设结论。”
- “请先保留相关材料，在正式核查结束前不要做额外传播。”

### 5. 对外稳口径

- “目前已确认的信息只有 X，其余内容还在核实。”
- “我们会在适当时点统一更新，现阶段不对未核实内容发表评论。”

## 避免用语

- “大家都知道你有问题”
- “你最好配合，不然后果自负”
- “上面已经决定了，只是通知你”
- “监管那边我们都打过招呼了”
- “你自己识相一点”

这些表达的问题是：容易失控、留把柄、升级对抗、暗示不当行为。

## 改写原则

遇到过猛原句时，优先改成：

1. 从人格判断改成组织判断
2. 从情绪宣泄改成事实与安排
3. 从口头威胁改成边界说明
4. 从模糊暗示改成明确但有限的表达
